> 快捷命令，查看当前需求的开发进度、任务级别和效能指标

---

## 使用方式

```
/dev-status
```

## 显示内容

### 基本信息
- 需求名称和ID
- 任务级别 (L1/L2/L3)
- 流程阶段数和确认点数
- 开始时间

### 阶段列表
- 各阶段状态标记
  - ✅ 已完成
  - 🔄 进行中
  - ⏳ 待开始
- 确认点标记
- 完成时间

### 效能指标
- 已耗时
- 预估剩余时间
- 确认次数
- 回退次数 (如有)

### 产物清单
- 已生成的文档
- 代码文件 (完成时)

### 关键决策 (如有)
- 技术决策记录

## AI贡献度量

显示AI生成代码的贡献度指标：

```
AI贡献度:
├── AI生成: 285行       # AI首次生成的代码行数
├── 人工编辑: 35行      # 人工修改的行数
├── 贡献率: 88%         # 1 - (人工编辑 / AI生成)
└── 质量: 高            # 基于编辑距离判断
```

### 贡献率判定

| 贡献率 | 评级 | 说明 |
|--------|------|------|
| ≥80% | 高 | AI生成质量高，少量调整 |
| 50-79% | 中 | 需要人工调整 |
| <50% | 低 | AI辅助有限 |

### 质量判定

基于编辑距离 (Edit Distance):
- **高**: 编辑距离 < 15%，AI生成代码质量高
- **中**: 编辑距离 15-30%，需要中等修改
- **低**: 编辑距离 > 30%，需要大量修改
