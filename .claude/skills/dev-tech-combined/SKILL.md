# 技术分析+方案 (Tech Combined)

> L2中等流程第2阶段：技术分析 → 轻量级方案 → 一次性确认

---

## 适用场景

- 新增表单字段类型
- 新增业务组件
- 常规功能开发
- 模块内扩展

---

## 执行流程

```
需求分析结果
     ↓
┌─────────────────────────────────────┐
│ 1. 快速技术分析                      │
│    - 定位涉及模块                    │
│    - 识别扩展点                      │
│    - 评估改动范围                    │
└─────────────────────────────────────┘
     ↓
┌─────────────────────────────────────┐
│ 2. 轻量级技术方案                    │
│    - 核心设计思路                    │
│    - 关键接口定义                    │
│    - 数据模型概要                    │
└─────────────────────────────────────┘
     ↓
┌─────────────────────────────────────┐
│ 3. 输出合并文档                      │
│    - 技术方案.md                     │
│    - 包含分析+设计                   │
└─────────────────────────────────────┘
     ↓
✓ 确认点 → 进入编码计划阶段
```

## 使用示例

```
需求分析结果:
- 需求: 为表单引擎添加AI按钮字段
- 涉及模块: 表单引擎(高), AI能力(中)

AI 执行技术分析+方案:
1. 技术分析:
   - 扩展点: BaseFormField → 新建AIButtonField
   - 改动: form-model新增字段类, form-engine新增处理器
   - 风险: 中(涉及AI调用)

2. 技术设计:
   - 核心思路: 继承BaseFormField，通过AI桥接层调用
   - 接口: AIButtonClickHandler
   - 数据模型: agentId, skillId, params

3. 输出: 技术方案.md

等待用户确认...
```
