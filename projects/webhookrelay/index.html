webhook-relay.js 
import { WebhookRelay } from '@webhookrelay/sdk';

const relay = new WebhookRelay({
  endpoint: 'https://api.webhookrelay.io',
  retries: 5,
  timeout: 30000
});

// Forward webhooks with automatic retry
relay.forward('stripe.payment', 'https://your-app.com/webhooks');
relay.forward('github.push', 'https://ci.your-app.com/hook');

// Get detailed logs and alerts
relay.onFailure((event) => {
  console.log('Webhook failed after 5 retries:', event);
});

Frequently Asked Questions

Why not just use Stripe/GitHub webhooks directly?

Native webhooks have no retry logic, limited logging, and no alerting. When they fail, you're blind. WebhookRelay adds the reliability layer you need.

How is this different from webhook services like Svix?

Svix is great for sending webhooks. We focus on receiving and forwarding them reliably. Think of us as a smart proxy between webhook sources and your app.

What happens if my server is down?

We queue webhooks and retry with exponential backoff. Once your server is back up, we deliver all pending webhooks. Nothing is lost.

Is there a free tier?

Yes! We'll offer a free tier with 1,000 webhooks/month for hobby projects. Perfect for testing and small apps.
