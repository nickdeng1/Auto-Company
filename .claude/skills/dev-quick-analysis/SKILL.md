# 快速分析 (Quick Analysis)

> L1简单流程第1阶段：快速理解需求 → 定位修改点 → 直接进入实现

---

## 适用场景

- Bug修复
- 配置调整
- 小功能改动
- 单文件修改

---

## 执行流程

```
需求描述
     ↓
┌─────────────────────────────────────┐
│ 1. 快速需求理解 (30秒)               │
│    - 提取核心意图                    │
│    - 识别关键词                      │
│    - 确定任务类型                    │
└─────────────────────────────────────┘
     ↓
┌─────────────────────────────────────┐
│ 2. 模块定位 (1分钟)                  │
│    - 根据关键词定位模块              │
│    - 查找相关文件                    │
│    - 确定修改范围                    │
└─────────────────────────────────────┘
     ↓
┌─────────────────────────────────────┐
│ 3. 输出分析摘要                      │
│    - 简要需求理解                    │
│    - 修改文件列表                    │
│    - 预估改动量                      │
└─────────────────────────────────────┘
     ↓
直接进入快速实现阶段 (无需确认)
```

---

## 输出模板

```markdown
## 快速分析

**需求理解**: [一句话描述]

**修改范围**:
- 文件1: path/to/file.java - [修改点描述]
- 文件2: path/to/file.java - [修改点描述]

**预估改动**: ~[N]行代码

**下一步**: 进入快速实现
```
